# API 重构计划

## 当前状态

项目的 API 重构已经从“建立 `/api/v1/*` 入口”进入“清理旧兼容层和收尾文档”阶段。

现状总结：
- 主业务接口已基本统一到 `/api/v1/*`
- 大量旧 `/api/*` 路由已物理删除
- 剩余旧入口主要集中在认证、上传和少量仍需兼容的路径
- 新增接口默认只应提供 `v1` 版本

## 已完成事项

### 已完成版本化并删除旧入口的模块
- `blog`
- `messages`
- `agent`
- `radar`
- `video`
- `comments`
- `user/social`
- `site/public`
- `friend-links`
- `categories`
- `tags`

### 已完成的配套工作
- 前端主调用已迁移到 `/api/v1/*`
- 旧路由删除前已补充弃用头和迁移文档
- `v1` 主入口已有独立测试覆盖

## 当前保留的非 `v1` 入口

这些入口仍保留是出于产品语义或迁移成本考虑，不是版本化遗漏：
- `/api/login`
- `/api/register`
- `/api/logout`
- `/api/upload`

## 后续原则

1. 新接口默认只新增 `/api/v1/*`
2. 不再为已删除的旧路径恢复兼容层
3. 路由重构优先做共享实现，再删旧 route 文件
4. 删除前必须满足：
   - 仓库内无旧路径调用
   - `v1` 不再转发旧 route 文件
   - 定向测试通过
   - `npm run type-check` 通过

## 下一步建议

1. 继续清理历史文档中的旧路径描述
2. 对仍保留的非 `v1` 入口补更明确的长期策略
3. 配合 Drizzle 迁移，继续减少 route 层重复逻辑
